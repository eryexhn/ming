# 1、TypeScript的介绍

  • 始于JavaScript，归于JavaScript

    TypeScript可以编译出纯净、简洁的JavaScript代码，并且可以运行在任何浏览器上、Node.js环境中和任何支持ECMAScript3（或更高版本）的JavaScript引擎中

  • 强大的类型系统

    类型系统允许JavaScript开发者在开发JavaScript应用程序时使用高效的开发工具和常用操作比如静态检查和代码重构

  • 先进的JavaScript

    TypeScript提供最新的和不断发展的JavaScript特性，包括那些ES5和未来提案中的特性，比如异步功能和decorators，以帮助建立健壮的组件

# 2、TypeScript的安装

全局安装TypeScript

npm install -g typescript

可检查是否安装成功
tsc -v

# 5、简单认识TypeScript

# 5.1、类型注解

给函数的参数person添加: string类型注解

function greeter (person: string) {
  return 'Hello, ' + person;
}

let user = 'Yee';

console.log(greeter(user));

TypeScript里的类型注解是一种轻量级的为函数或变量添加约束的方式。这代码中我们希望greeter函数接收一个字符串参数。然后尝试把greeter的调用改成传入一个数组

function greeter (person: string) {
  return 'Hello, ' + person;
}

let user = [0, 1, 2];

console.log(greeter(user));

自动编译就会产生一个错误

error TS2345: Argument of type 'number[]' is not assignable to parameter of type 'string'.

要注意的是尽管有错误，greeter.js文件还是会被创建。就算你的代码里有错误，你仍然可以使用TypeScript。但在这种情况下，TypeScript会警告你代码可能不会按预期执行

类似的，尝试删除greeter调用所有参数。TypeScript会告诉你使用了非期望个数的参数调用了这个函数。在这两种情况中，TypeScript提供了静态的代码分析，它可以分析代码结构和提供的类型注解

# 5.2、基础静态类型

基础静态类型只需要在声明变量的后边加上一个:号，然后加上对应的类型。

基础类型有：string、number、boolean、null、undefined、symbol、void、any

void和undefined差不多

any为任意一个类型

let str: string = '这是一段字符串';
let num: number = 20;
let bol: boolean = true;
let nul: null = null;
let und: undefined = undefined;
let sym: symbol = Symbol();
let an: any = 1;

# 5.3、对象类型

// 对象类型
const people: {
  name: string,
  age: number
} = {
  name: '张三',
  age: 18
}
console.log(people.name);

// 数组类型
// 都是字符串类型的数组
const peoples: string[] = ['张三', '李四', '王五'];

// 类类型
class Person {};
const dajiao: Person = new Person();

// 函数类型
// 函数的返回值为字符串类型
const func: () => string = () => {
  return '张三';
}

# 5.4、接口

示例中使用接口来描述一个拥有firstName和lastName字段的对象。在TypeScript里，只在两个类型内部的结构兼容，那么这两个类型就是兼容的。

interface Person {
  firstName: string
  lastName: string
  age?: number  // ?:表示该字段可有可没有
}

function greeter (person: Person) {
  return 'Hello, ' + person.firstName + ' ' + person.lastName;
}

let user = {
  firstName: 'Yee',
  lastName: 'Huang',
  // 当这里出现接口定义中为定义的字段，则会在上面函数中调用时报错
  sex: 'nan'
}

console.log(greeter(user));

# 5.5、类

创建一个User类，带有一个构造函数和一些公共字段，因为类的字段包含了接口所需要的字段

class User {
  fullName: string
  firstName: string
  lastName: string

  constructor (firstName: string, lastName: string) {
    this.firstName = firstName;
    this.lastName = lastName;
    this.fullName = firstName + ' ' + lastName;
  }
}

interface Person {
  firstName: string
  lastName: string
}

function greeter (person: Person) {
  return 'Hello, ' + person.firstName + ' ' + person.lastName;
}

let user = new User('Yee', 'Huang');

console.log(greeter(user));

重新运行tsc greeter.ts，就会发现TypeScript中的类只是一个语法糖，本质上还是JavaScript函数的实现

# 6、基础类型

TypeScript支持与JavaScript几乎相同的数据类型

TypeScript中变量一开始是什么类型，那么后期赋值的时候，只能用这个类型的数据，是不允许用其他类型的数据赋值给当前的这个变量中

# 6.1、布尔值

最基本的数据类型就是简单的true/false值

// 布尔类型  boolean
// 基本语法
// let 变量名: 数据类型 = 值;
let isDone: boolean = false;
isDone = true;
// isDone = 2 // error

# 6.2、数字

和JavaScript一样，TypeScript里的所有数字都是浮点型。这些浮点数的类型都是number。除了支持十进制和十六进制字面量，TypeScript还支持ECMAScript2015中引入的二进制和八进制字面量

// 数值类型  number
let a1: number = 10; // 十进制 10
let a2: number = 0b1010;  // 二进制 10
let a3: number = 0o12; // 八进制 10
let a4: number = 0xa; // 十六进制 10

# 6.3、字符串

使用双引号（"）或单引号（'）表示字符串

// 字符串类型  string
let name: string = 'tom';
name = 'jack';
// name = 12 // error
let age: number = 12;
const info = `My name is ${name}, I am ${age} years old!`;

# 6.4、undefined和null

TypeScript中，undefined和null两者各自有自己的类型分别叫做undefined和null。它们本身的类型用处不是很大

let u: undefined = undefined;
let n: null = null;

默认情况下null和undefined是所有类型的子类型。就是说你可以把null和undefined赋值给number类型的变量。（不在ts的严格模式下strict: false时）

# 6.5、数组

TypeScript像JavaScript一样可以操作数组元素。有两种方式可以定义数组，第一种，可以在元素类型后面接上[]，表示由此类型组成的一个数组

let list1: number[] = [1, 2, 3];

// 下面的方式可以定义多种类型再数组中，但是不能确定元素的位置是什么类型
let arr1: (number | string)[] = ['10', 20, 30, 40, 50, 60];
console.log(arr1);

第二种方式就是使用数组泛型，Array<元素类型>

let list2: Array<number> = [1, 2, 3];

// 数组定义方式1
// let 变量名: 数据类型[] = [值1, 值2, 值3]
let arr1: number[] = [10, 20, 30, 40, 50, 60];
console.log(arr1);

// 数组定义方式2：泛型的写法
// 语法：let 变量名: Array<数据类型> = [值1, 值2, 值3]
let arr2: Array<number> = [100, 200, 300];
console.log(arr2);

// 注意问题:数组定义后，里面的数据类型必须和定义数组的时候的类型是一致的，否则有错误提示信息，也不会编译通过

# 6.5.1、数组中基本类型的定义

一般数组类型的定义

定义一个简单的数组类型，比如数字类型的数组

const numberArr = [1, 2, 3];

加上类型注解

const numberArr: number[] = [1, 2, 3];

同样的字符串数组

const stringArr: string[] = ['a', 'b', 'c'];

以及undefined数组

const undefinedArr: undefined[] = [undefined, undefined];

单类型数组，就是说可以定义任意类型的数组

多种类型数组的注解

const arr: (number | string)[] = [1, 'string', 2];

# 6.5.2、数组中对象类型的定义

const arr: { name: string, age: number }[] = [
  {
    name: '张三',
    age: 18
  },
  {
    name: '李四',
    age: 20
  }
];

类型别名

定义别名的时候要以type关键字开始

type Lady = { name: string, age: Number };

const arr: Lady[] = [
  {
    name: '张三',
    age: 18
  },
  {
    name: '李四',
    age: 20
  }
];

class类定义别名

class Madam {
	name: string;
	age: number;
}

const arr: Madam[] = [
  {
    name: '张三',
    age: 18
  },
  {
    name: '李四',
    age: 20
  }
];

# 6.6、元组Tuple

元组类型允许表示一个已知元素数量和类型的数组，各元素的类型不必相同。比如，你可以定义一对值分别为string和number类型的元组

let t1: [string, number]
t1 = ['hello', 10] // OK
t1 = [10, 'hello'] // Error

// 元组类型：在定义数组的时候，类型和数据的个数一开始就已经一一对应
let arr3: [string, number, boolean] = ['小甜甜', 100, true];
console.log(arr3);
// 注意问题：元组类型在使用的时候，数据的类型的位置和数据的个数，应该和在定义元组的时候的数据类型及位置应该是一致的

当访问一个已知索引的元素，会得到正确的类型：

console.log(t1[0].substring(1)) // OK
console.log(t1[1].substring(1)) // Error, 'number' 不存在 'substring' 方法

# 6.7、枚举

enum类型是对JavaScript标准数据类型的一个补充。使用枚举类型可以为一组数值赋予友好的名字

// 枚举类型
// 枚举里面的每个数据值都可以叫元素，每个元素都有自己的编号，编号是从0开始的，依次的递增加1
enum Color {
    red,
    green,
    blue
}

// 定义一个Color的枚举类型的变量来接收枚举的值
let color: Color = Color.red;
console.log(color); // 0
console.log(Color.red, Color.green, Color.blue);// 0 1 2
console.log(Color[2]); // blue

默认情况下，从0开始为元素编号。也可以手动的指定成员的数值。例如，我们将上面的例子改成从1开始编号

enum Color { Red = 1, Green, Blue };
let c: Color = Color.Green; // 2

# 6.12、类型断言

通过类型断言这种方式可以告诉编译器，"相信我，我知道自己在干什么"。类型断言好比其他语言里的类型转换，但是不进行特殊的数据检查和解构。它没有运行时的影响，只是在编译阶段起作用。TypeScript会假设你，已经进行了必须的检查。

类型断言有两种形式。其一是尖括号语法，另一个为as语法

// 类型断言：告诉编译器，我知道我自己是什么类型，也知道自己在干嘛
// 类型断言的语法方式1: (<类型>变量名)
// 类型断言的语法方式2: (值 as 类型)
function getString(str: number | string): number {
    // return str.toString().length;
    // 如果str本身就是string类型，那么是没有必要调用toString()方法的

    // str.length存在吗？若存在则str是string类型的，不存在则str是number类型的
    return (<string>str).length ? (str as string).length : str.toString().length;

}

console.log(getString('123456'));

  name: string
  age: number
  sex?: string

# 7.3、函数类型

    return source.search(subString) > -1;

# 8.2、继承

在TypeScript中，我们可以使用常用的面向对象模式。基于类的程序设计中一种最基本的模式是允许使用继承来扩展现有的类

// 继承：类与类之间的关系
// 继承后类与类之间的叫法：
// A类继承了B类，那么此时A类叫子类，B类叫基类(父类)
// 子类 ---> 派生类
// 基类 ---> 超类(父类)
// 一旦发生了继承的关系，就出现了父子类的关系

// 定义一个类，做为基类(父类)
class Person {
    // 定义属性
    name: string; //名字
    age: number; //年龄
    gender: string; //性别
    // 定义构造函数
    constructor (name: string = '小明', age: number = 18, gender: string = '男') {
        //更新属性数据
        this.name = name;
        this.age = age;
        this.gender = gender;
    }
    // 定义实例方法
    sayHi (str: string): void {
        console.log(`我是${this.name}, ${str}`);
    }
}

// 定义一个类，继承自Person
class Student extends Person {
    // 定义构造函数
    constructor (name: string, age: number, gender: string) {
        // 调用的是父类中的构造函数，使用的是super
        super(name, age, gender)
    }

    // 可以调用父类中的方法
    sayHi(): void {
        console.log('我是学生类中的sayHi方法');
        // 调用父类中的sayHi方法
        super.sayHi('hello world');
    }
}

// 实例化 Person
const person = new Person('大明明', 89, '男');
person.sayHi('嘎嘎');

// 实例化 Student
const student = new Student('小红', 18, '女');
student.sayHi();

// 总结：类和类之前如果要有继承关系，需要使用功能extends关键字
// 子类中可以调用父类中的构造函数，使用的是super关键字(包括调用父类中的实例方法，也可以使用super)
// 子类中可以重写父类中的方法

# 10.2、泛型接口

在定义接口时吗，为接口中的属性或方法定义泛型类型

在使用接口时，再指定具体的泛型类型

// 泛型接口:在定义接口时，为接口中的属性或方法定义泛型类型，在使用接口时，再指定具体的泛型类型

// 需求:定义一个类，用来存储用户的相关信息(id, name, age)
// 通过一个类的实例对象调用相关的方法可以添加多个用户信息对象，调用getUserId方法可以根据id获取某个指定的用户信息对象
interface IBaseCRUD<T> {
    data: Array<T>;
    add: (t: T) => T;
    getUserId: (id: number | undefined) => T | undefined
}

// 定义一个用户信息的类
class User {
    id?: number; // id // number | undefined
    name: string; // 名字
    age: number; //年龄

    constructor (name: string, age: number) {
        this.name = name;
        this.age = age;
    }
}

// 定义一个类，可以针对用户的信息对象进行增加及查询的操作
// CRUD ----> create, read, update, dalete
class UserCRUD implements IBaseCRUD<User> {
    // 用来保存多个User类型的用户信息对象
    data: Array<User> = [];
    // 用来存储用户信息对象的
    add (user: User): User {
        user.id = Date.now() + Math.random();
        this.data.push(user);
        return user;
    }
    // 根据id查询指定的用户信息对象
    getUserId (id: number | undefined): User | undefined {
        return this.data.find(user => user.id === id);
    }
}

// 实例化对象
const userCRUD: UserCRUD = new UserCRUD();
userCRUD.add(new User('jack', 20));
userCRUD.add(new User('tom', 25));
const { id } = userCRUD.add(new User('lucy', 23));
userCRUD.add(new User('rousi', 21));
console.log(userCRUD.data);
// 根据id查询用户信息的对象
const user = userCRUD.getUserId(id);
console.log(user);

console.log(g1.add(g1.defaultValue, 20));

# 10.4、泛型约束

若直接对一个泛型参数取length属性，会报错。因为这个泛型根本就不知道它有这个属性

// 没有泛型约束
function fn <T>(x: T): void {
  // console.log(x.length)  // error
}

泛型约束来实现

// 泛型约束

// 定义一个接口，用来约束将来的某个类型中需要有length这个属性
// 相当于在泛型约束的某个类型中必须要有length这个属性
interface ILength {
    // 接口中有一个属性length
    length: number
}
function getLength<T extends ILength> (x: T): number {
    return x.length;
}

console.log(getLength<string>('abc')); // 3
// console.log(getLength<number>(2)); // 类型“number”不满足约束“ILength”。

# 11.1、声明文件

当使用第三方库时，我们需要引用它的声明文件，才能获得对应的代码补全、接口提示等功能

假如使用第三方库jquery，一种常见的方式是在html中通过<script>标签引入jquery，然后就可以使用全局变量$或jquery

但是在ts中，编译器并不知道$或jquery是什么东西

/* 
	当使用第三方库时，我们需要引用它的声明文件，才能获得对应的代码补全、接口提示等功能。
	声明语句: 如果需要ts对新的语法进行检查, 需要要加载了对应的类型说明代码
  	declare var jQuery: (selector: string) => any;
	声明文件: 把声明语句放到一个单独的文件（jQuery.d.ts）中, ts会自动解析到项目中所有声明文件
	下载声明文件: npm install @types/jquery --save-dev
*/

jQuery('#foo');
// ERROR: Cannot find name 'jQuery'.

这时，需要使用declare var来定义类型

declare var jQuery: (selector: string) => any;

jQuery('#foo');

declare var并没有真的定义一个变量，只是定义了全局变量jquery类型，仅仅会用于编译时的检查，在编译结果中会被删除

一般声明文件都会单独写成一个xxx.d.ts文件

创建jquery.d.ts，将声明语句定义其中，TS编译器会扫描并加载项目中所有的TS声明文件

declare var jQuery: (selector: string) => any;

但这个声明文件，只定义了一个jQuery这个成员，需要jQuery中的其他成员还有手动进行定义类型

所以要使用第三方库的声明文件

比如

npm install @types/jquery --save-dev

下载后就可以不用declare var去定义类型

直接使用正常的引入方式即可

import jQuery from "jquery";
jQuery('');

很多的第三方库都定义了对应的声明文件库，库文件名一般为@types/xxx，可以在https://www.npmjs.com/package/package进行搜索

有些第三库在下载时就会自动下载对应的声明文件库(比如：webpack)，有的可能就需要单独下载(比如：jQuery/react)
